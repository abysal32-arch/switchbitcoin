## Findings

1. **F-E1 (product, FIXED `6b02f01`)** — node restart permanently+silently
   wedged serve/watch. Fix: 401 → cookie re-read + single retry (unchanged
   file = clean explicit 401/403 error, no hammering); per-call reachability
   tracking (`node_online()` — node-level RPC errors stay "online"); loud
   ALARM/recovery edges in serve AND watch; TESTER-GUIDE §9 row ("restarting
   bitcoind is SAFE"). 4 regression tests (canned-transport flap + real-HTTP-
   stub cookie rotation).
2. **F1 (product, cosmetic — FIXED this close-out)** — serve stderr ~388
   KB/day (measured: 307,401 B over ~19 h), dominated by re-logging
   `settled — nothing to drive` for every terminal record every recovery
   pass (~50 s). Fix: the serve loop's periodic recovery pass now skips
   terminal `Settled` ticks (named `serve_pass_skips_tick` + regression test
   that the scope stays EXACTLY Settled — a refund/rebroadcast tick must
   never be dropped); the one-shot `recover` still logs Settled as useful
   confirmation. Committed with this close-out (does NOT re-cut the live
   download — 6b02f01cf stands; the next tester package picks it up).
3. **F2 (harness, not product)** — unattended scheduled check-ins stalled on
   permission prompts (3 fires: 01:01Z, 08:00Z one-shot, 13:0xZ). Every
   payload was covered by manual sweeps / pre-fired automation. Not a product
   issue; noted so the tester round doesn't depend on unattended cron.
4. **F3 (environment)** — PC clock ~1.9 h behind (w32time service stopped,
   start needs admin). Broke min-difficulty mining eligibility (the miner
   ran its full 12 h cap, 04:38→16:38Z, without a block) and skews all
   wall-clock data. STILL OPEN at close-out — needs Joe's admin action
   (`Start-Service w32time; w32tm /resync /force`). Not a product bug; a
   host-config gate on the funded-swap leg.
